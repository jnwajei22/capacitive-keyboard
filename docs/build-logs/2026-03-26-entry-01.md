Entry 1: Hierarchical schematic restructure and core power/MCU architecture
Date:

2026-03-27

Change made:

Reorganized the keyboard schematic from a cluttered single-sheet design into hierarchical sheets and substantially wired the major subsystem pages. The sheet structure now consists of:

POWER
MCU_CORE
KEY_MATRIX
DISPLAY
RGB

The POWER sheet was built out around the selected production parts and rail architecture:

USB-C connector: GCT USB4105-GF-A
USB ESD protection: TI TPD2E2U06DRLR
Charger / power-path IC: TI BQ24075RGTT
Fuel gauge: MAX17048G+T10
3.3 V LDO: TI TPS7A2033PDBVR
5 V boost converter: MIC2875-5.0YMT-TR

The power tree was defined and wired as:

VBUS = raw USB 5 V from the USB-C connector
+BATT = battery positive rail
SYS_OUT = charger/power-path system rail
+3V3 = LDO output from SYS_OUT
BOOST_5V = boost converter output from SYS_OUT
DISP_5V and LED_5V = branch rails sourced from BOOST_5V

The USB-C sink configuration was wired correctly for USB 2.0 operation with both CC pins using 5.1 kΩ pulldowns, USB D+ / D- paired across both plug orientations, VBUS broken out, and shell/ground tied to GND. The USB ESD device was added on D+ / D-.

The charger block was corrected against the BQ24075 datasheet:

CE tied low
SYSOFF tied low
TS changed to a single 10 kΩ resistor to GND
EN1 tied low
EN2 tied to VBUS
ILIM and ISET set with external resistors
BAT, IN, and OUT decoupling added

The fuel gauge block was corrected and finalized for MAX17048 specifically:

VDD tied to +BATT
CELL left NC
0.1 µF local bypass added
SDA/SCL pulled up to +3V3
FG_ALERT given an external pull-up to +3V3
QSTRT, CTG, GND, and EP tied to GND

The LDO block was wired with:

IN = SYS_OUT
EN = SYS_OUT
OUT = +3V3
4.7 µF input and output capacitors

The boost block was added using the MIC2875 fixed 5 V configuration:

IN = SYS_OUT
EN = SYS_OUT
SW connected through a 1 µH inductor to SYS_OUT
OUT and OUTS tied together to BOOST_5V
PG tied low since unused
AGND, PGND, and EXP tied to GND
4.7 µF input capacitor and 22 µF output capacitor added

The MCU_CORE sheet was then largely completed around the Raytac MDBT50Q-1MV2 module:

VDD tied to +3V3
VBUS tied to raw VBUS
VDDH and DCCH intentionally left NC for the <3.6 V supply case
SWDIO, SWCLK, RESET, USB_D+, and USB_D- wired
external RESET pull-up added
status LED resistor increased to 1 kΩ
I2C and UART assignments moved off low-frequency-only pins
FG_ALERT placed on an acceptable low-frequency GPIO
matrix row/column nets assigned
5 RGB data nets assigned

The KEY_MATRIX sheet was kept as a full explicit matrix rather than being abstracted into blocks. It currently implements:

6 rows x 16 columns
96 switch positions
one 1N4148W diode per key
Fn included as a normal matrix key
row-scan / column-read convention chosen and documented

The RGB sheet was built as five separate chains of ten WS2812B-2020 LEDs:

each chain has its own LED data input
each chain includes a 330 Ω series data resistor
each LED has a 0.1 µF local decoupling capacitor
each chain has a bulk capacitor
the design was intentionally kept as 5 parallel chains of 10 instead of 1 chain of 50 for easier testing and fault isolation

The DISPLAY sheet was kept intentionally simple around a 4-pin connector:

DISP_5V
GND
display UART TX/RX pair
Why:

The original flat schematic had become messy and harder to reason about. Breaking the design into hierarchical subsystem sheets made the project easier to review, easier to route later, and much less likely to hide wiring mistakes.

The POWER architecture needed to be made explicit because the design includes:

USB input
battery operation
3.3 V logic
5 V display power
5 V RGB power

Using the BQ24075 as the power-path manager simplified USB/battery handoff and created a proper managed system rail. The LDO and boost converter were then used for the actual voltage conversion tasks.

The MCU pinout also needed cleanup because the Raytac module has low-frequency-only GPIO that are not suitable for UART or I2C. Moving those interfaces onto normal GPIO avoided a real hardware mistake before layout.

Keeping RGB as 5 independent chains improves bring-up and debugging. If one chain fails, the other four still work, and each segment can be tested separately.

Result:

The schematic architecture is now coherent and mostly validated at the subsystem level. POWER, KEY_MATRIX, DISPLAY, RGB, and most of MCU_CORE have been defined with real parts and corrected against datasheets. Major pin-function mistakes were caught before layout, especially around the Raytac module’s low-frequency-only GPIO. The project is now at the point where the top-level sheet can be cleaned into a pure hierarchical interconnect page and the design can move toward final schematic review and PCB layout.
